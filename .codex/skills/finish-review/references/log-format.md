# Review Log Format

Use `reviews/<github-handle>.md` as the canonical location.

## Structure

- top-level student heading
- `Profile` section
- one section per week

## Profile section

Capture long-lived calibration such as:

- base branch pattern
- review style or bar
- recurring strengths
- recurring watch points

Do not overstate certainty. Keep recurring themes concise.

## Weekly section

Record:

- PR link
- branch pair
- PR title when useful
- assignment focus
- actual comments left
- follow-up notes for future reviews

If merged/open state is known from the current fetch, it can be noted briefly, but this is not the main purpose of the log.

## Source-of-truth rule

- Only actual posted GitHub review comments go into the log.
- Draft suggestions from earlier conversation do not count unless the user really posted them.

## Continuity rule

Write the log so next week's review can use it as calibration context:

- what was praised
- what was pointed out
- what should be watched next time
