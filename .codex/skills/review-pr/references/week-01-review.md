# Week 1 Review Notes

Week 1 covers:

- React setup
- basic UI implementation from Figma
- component split
- import/export
- JSX
- curly braces in JSX

Not yet covered:

- `react.dev` rendering lists

## Review implications

- Do not require `map`-based list rendering for Week 1 unless the user explicitly wants a stricter review.
- Semantic HTML comments can still be valid, but prioritize the more direct structure issue first.
- For first React submissions, value:
  - successful component split
  - clear import/export structure
  - visible attempt to match the Figma layout

## Common valid comments

- leftover Vite starter CSS or template artifacts
- unused imported files or styles
- obvious semantic issues like `ul/li` for a list, if worth mentioning
- heading semantics when it is a clear improvement and not lower priority than list semantics
- file naming, import casing, or role mismatches

## Common comments to skip

- forcing future-week abstractions
- production-grade architecture feedback on very small beginner assignments
- nitpicks that do not help the student progress this week
