# Objective Format

Write files to `weekly-objectives/week-XX.md`.

## Required sections

- `# Week XX — <title>`
- `## Source`
- `## Reading Objectives`
- `## Practice Objectives`
- `## Required Review Focus`
- `## Optional Review Comments`
- `## Skip Comments For This Week`
- `## Critical Issues Override`
- `## Notes`

## Writing rules

- Keep the file concise.
- Write for reviewer use, not for students.
- Focus on what changes review decisions.
- Prefer short bullets over long prose.

## Important rule

The objective file should guide scope, not silence important issues.

Use `Critical Issues Override` to explicitly preserve comments for:
- broken implementations
- path/import problems
- confusing starter leftovers
- ineffective code that matters to the submission
