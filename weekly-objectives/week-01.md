# Week 01 — React 세팅 + 기본 UI

## Source

- Notion curriculum page: [Week 1 — React 세팅 + 기본 UI](https://www.notion.so/31b793efb801813eb57fdd78bffa84d9)

## Reading Objectives

- Understand what a React component is
- Understand importing and exporting components
- Understand basic JSX syntax
- Understand using curly braces inside JSX for simple values

## Practice Objectives

- Initialize the React assignment environment
- Implement the Week 1 Figma layout
- Split the UI into `TodoHeader`, `TodoCard`, `TodoList`, and `App`
- Compose the screen through import/export
- Use JSX and curly braces for visible values without requiring repeated list rendering

## Required Review Focus

- Component split is clear and matches the assignment
- Import/export structure is understandable
- The layout visually follows the Figma design
- Spacing, size, typography, color, radius, and shadow are close to the Figma
- The page structure is tidy enough for a first React UI

## Optional Review Comments

- Semantic HTML improvements such as `ul` / `li` when the screen is clearly a list
- Small cleanup of leftover starter files or styles
- Slightly clearer naming when it directly affects readability

## Skip Comments For This Week

- Do not require list rendering with `map` if the student has not done it yet
- Do not over-push production-level architecture or abstraction
- Do not demand patterns that belong to later React learning stages
- Do not over-review fixed-width layout if it follows the provided Figma closely

## Critical Issues Override

- Still comment on clearly broken or misleading assignment implementations
- Still comment on import or path issues that may fail in real environments
- Still comment on obvious leftover starter/template artifacts when they make the submission confusing
- Still comment on clearly ineffective or unused code when it affects the actual submission quality

## Notes

- `읽을 거리` should limit concept-level expectations.
- `실습` should drive the main review comments.
- If a student voluntarily uses a later concept correctly, it can be noted positively without making it a requirement for everyone.
